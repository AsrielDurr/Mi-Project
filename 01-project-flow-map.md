# M1 AI推荐模块 · 项目流程图

## 项目定位

项目名称：M1 AI推荐模块（AI Course Recommendation）

项目形态：FastAPI / Python HTTP 服务

技术基线：Python 3、纯 dataclass（无 pydantic 依赖）、JSON 种子数据、HTTP API

输入：
- 学生画像（StudentProfile：student_id, goal, skills, available_times, completed_course_ids, enrolled_course_ids）

输出：
- 推荐结果（RecommendationResponse：trace_id, source, model, fallback_reason, recommendations[]）
- 前端事件（CourseSelectedEvent）

推荐逻辑：
- 优先调用 MiMo 模型生成推荐
- 模型不可用时降级为本地规则推荐
- 规则推荐基于：目标匹配、先修课程检查、时间冲突检测、名额可用性

非目标：
- 选课决策（M2 负责）
- 候补队列管理（M3 负责）
- 用户登录 / 权限管理
- 数据库存储（纯内存 + JSON 种子）
- WebSocket 实时推送

# 关键交接规则

-   产品 → 设计：所有 P0 功能必须能够设计实现；每项验收标准必须可验证。
-   设计 →
    研发：每项设计必须对应开发任务；推荐结果必须符合共享契约 v1.0.0。
-   研发 → 测试：每个功能必须提供验证命令；输出结果必须可重复。
-   测试 → QA：每项测试必须提供测试命令、测试样例、JSON 响应；所有结论必须可复现。
-   QA → 产品：任一 Gate
    未通过必须退回对应负责人；修复完成后重新执行完整测试。

# 项目关键断点

  断点                         后果                   修复动作
  ---------------------------- ---------------------- ---------------------------
  PRD 缺少验收标准             无法判断是否完成       补充可观察验收结果
  设计超出非目标               项目范围失控           调整设计或更新 PRD
  开发没有验证命令             无法证明完成           为每个功能补充验证命令
  测试无法复现                 Bug 难以定位           固定学生画像、课程数据、环境
  QA 只有通过/失败             无法追踪修复           明确退回对象与复验流程
  推荐结果不符合契约           前端无法对接           对齐 domain.schema.json 字段

# AI 参与点与人工责任

AI 可参与：

-   编写 PRD 草稿
-   比较设计方案
-   生成代码模板
-   生成测试用例
-   生成推荐规则
-   编写项目文档

人工负责：

-   确认需求范围
-   审核设计取舍
-   验证推荐结果合理性
-   审核测试证据
-   最终 QA 准出

AI 输出仅作为辅助，不可直接作为项目通过依据；最终结果必须由本地 HTTP
API 执行和固定学生画像验证。

# 完成定义（Definition of Done）

-   八份文档内容一致且无冲突；
-   M1 服务能正确处理推荐请求；
-   推荐结果符合共享契约 v1.0.0 格式；
-   MiMo 模型调用和降级方案均能正常工作；
-   测试可重复执行；
-   QA Gate 均有完整证据支撑；
-   AI 使用符合 token-strategy.md 的最小上下文原则。
