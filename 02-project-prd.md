# M1 AI推荐模块 PRD

## 1. 项目概述

### 项目名称

M1 AI推荐模块（AI Course Recommendation）

### 项目定位

一个运行在后端的 Python HTTP 服务，根据学生画像（目标、技能、已选课程、时间偏好）推荐合适的课程，优先使用 MiMo 大模型，降级时使用本地规则引擎。

### 技术基线

- Python 3
- 纯 dataclass（无 pydantic 依赖）
- HTTP API（JSON 格式）
- JSON 种子数据
- MiMo LLM（预留接口）

### 项目目标

帮助学生发现适合自己的课程，提升选课效率；为后续 M2（规则选课）和 M3（候补追溯）模块提供推荐数据源。

---

# 2. 用户与使用场景

## 目标用户

### 学生

填写个人画像后获取课程推荐，快速了解适合自己的课程方向。

### 前端应用

调用推荐 API 获取推荐结果，展示给学生并触发 `course-selected` 事件。

### 系统集成

M2 模块在学生选课前可调用推荐接口，获取推荐课程列表作为选课参考。

---

## 使用场景

### 场景一：学生请求推荐

学生填写目标（人工智能/软件开发）、已完成课程、时间偏好等信息，前端调用 `POST /api/recommend`，获取推荐课程列表。

---

### 场景二：MiMo 模型不可用时降级

MiMo API 超时或不可用时，系统自动降级为本地规则推荐，返回 `source: "FALLBACK"` 和 `fallback_reason`。

---

### 场景三：学生选择推荐课程

学生在前端点击推荐课程时，前端发出 `course-selected` 事件，M1 记录用于后续分析。

---

# 3. 项目目标

本项目需要完成以下目标：

- 接收学生画像（StudentProfile）
- 执行推荐逻辑（规则引擎 + MiMo 降级）
- 返回符合契约的推荐结果（RecommendationResponse）
- 支持先修课程检查
- 支持时间冲突检测
- 支持名额可用性检查
- 基于目标的课程评分
- 输出 trace_id 用于追溯
- 提供前端事件接口

---

# 4. 非目标

本项目不包含以下功能：

- 选课决策（M2 负责）
- 候补队列管理（M3 负责）
- 用户登录 / 权限管理
- 数据库存储（纯内存 + JSON 种子）
- WebSocket 实时推送
- 课程管理（增删改查）
- 学生管理
- 真实 MiMo 模型调用（预留接口）

# 5. 功能需求

## R1：接收学生画像

### 优先级

P0

### 描述

接收符合契约的 StudentProfile 对象。

### 验收标准

- 请求体包含 student_id, goal, skills, available_times, completed_course_ids, enrolled_course_ids
- 字段类型和必填性符合 domain.schema.json
- 缺少必填字段返回 422 错误

---

## R2：规则推荐（降级方案）

### 优先级

P0

### 描述

当 MiMo 模型不可用时，使用本地规则引擎生成推荐。

### 验收标准

- 基于目标匹配评分（AI方向 → AI课程加分，软件开发 → 开发课程加分）
- 检查先修课程是否满足
- 检查时间冲突
- 检查名额可用性
- 返回 score 0-100 的推荐列表
- source 为 "FALLBACK"

---

## R3：MiMo 模型推荐（预留）

### 优先级

P1

### 描述

调用 MiMo 模型生成个性化推荐。

### 验收标准

- 模型调用成功时返回 source="MODEL"，model 为模型名称
- 模型调用失败时自动降级为规则推荐
- 返回 trace_id 用于追溯

---

## R4：返回推荐结果

### 优先级

P0

### 描述

返回符合 RecommendationResponse 契约的 JSON。

### 验收标准

- 包含 trace_id, source, model, prompt_version, fallback_reason, recommendations
- 每个 recommendation 包含 course_id, score, reason, uncertainty
- recommendations 数组至少包含 1 项
- 字段命名符合 snake_case（契约要求）

---

## R5：先修课程检查

### 优先级

P0

### 描述

推荐时检查学生是否满足课程先修要求。

### 验收标准

- 学生已完成或正在修读先修课程 → 可推荐
- 缺少先修课程 → 不推荐该课程
- 无先修要求 → 可推荐

---

## R6：时间冲突检测

### 优先级

P0

### 描述

推荐时检查新课程与已选课程是否存在时间冲突。

### 验收标准

- 同一天同一时间段 → 冲突，不推荐
- 不同天或不重叠时间段 → 无冲突，可推荐

---

## R7：前端课程选择事件

### 优先级

P2

### 描述

接收前端发出的课程选择事件。

### 验收标准

- 接收 CourseSelectedEvent（studentId, courseId, recommendationTraceId）
- 返回 200 状态码
- 事件格式符合 frontend-events.ts（camelCase）

---

## R8：健康检查

### 优先级

P1

### 描述

提供服务健康检查端点。

### 验收标准

- GET /health 返回 200
- 响应包含 status, module, version

---

# 6. 优先级（P0 / P1 / P2）

## P0（必须完成）

- 接收学生画像（R1）

- 返回推荐结果（R4）
- 先修课程检查（R5）
- 时间冲突检测（R6）

## P1（应该完成）

- MiMo 模型推荐预留（R3）
- 健康检查（R8）

## P2（可优化）

- 前端课程选择事件（R7）
- 更精细的评分算法
- 推荐结果缓存
- 多语言支持

# 7. AI 参与边界

AI 可以协助：

- 编写 PRD；
- 生成代码模板；
- 生成测试样例；
- 生成推荐规则；
- 生成文档。

- 确认需求；
- 审核设计；
- 验证推荐结果合理性；
- 审核测试证据；
- 最终验收。

AI 输出不能直接作为验收依据，必须由本地 API 执行结果验证。
