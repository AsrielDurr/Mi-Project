# 验证计划、用例结果与风险

## 1. 验证目标

- 证明模型输出受到目录和结构约束。
- 证明AI不能绕过规则。
- 证明满员进入候补而不是规则失败。
- 证明第一名失效后继续处理第二名。
- 证明三个模块共享状态和Trace。
- 证明前端展示与后端状态一致。

## 5. 建议补充的边界测试

- 已处理候补再次重算。
- 两个空位和三名候补。
- 相邻课程与重叠一分钟的时间边界。
- 非法枚举、额外字段和空ID不写状态。
- 快速切换教师课程的请求竞态。
- 连续释放名额的防误操作策略。
- 仓库、响应、Trace和dist中的Key扫描。

这些项目用于增强边界证据，不改变当前两天MVP已经集成的结论。

## 6. 真实MiMo证据要求

运行`backend/scripts/verify_real_mimo.py`后，保存不包含Key的截图或脱敏记录。证据至少显示：

- `REAL_MIMO_OK`或页面`MODEL`来源。
- 实际模型名称。
- 至少一门目录内课程。
- `trace_id`。
- 日期和对应commit。
